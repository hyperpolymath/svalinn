= Rokur gate + ops docs, ported from the retired ReScript copy

`hyperpolymath/stapeln`'s `container-stack/svalinn/` was a second, diverged
copy of this repo — written in ReScript (now banned estate-wide) rather than
this repo's AffineScript/TypeScript line. That copy had two things this repo
never got: a working Rokur secrets-gate integration, and deployment/security
hardening docs. The nested copy has been retired; these are what's worth
keeping from it.

* `ROKUR-GATE-CONTRACT.adoc` — the Rokur gate's exact request/response
  behaviour, written from reading the (now-retired) ReScript source, so it
  can be reimplemented here without re-deriving the semantics. Start here.
* `API.adoc`, `DEPLOYMENT.adoc`, `TESTING.adoc` — carried over as-is from the
  ReScript copy. They describe that implementation's specifics (file paths,
  build commands, module names) and **will need editing** to match this
  repo's actual AffineScript/TypeScript structure before they're accurate —
  kept here for the operational knowledge (firewalld/ModSecurity/OpenLiteSpeed
  hardening steps, test strategy) rather than as ready-to-follow docs.
* `../../deployment/security/` — the firewalld zones, ModSecurity config, and
  OpenLiteSpeed config referenced by `DEPLOYMENT.adoc`. These are
  language-independent infra config and should need no changes.
* `affinescript-wasm-ctor-link.patch` — a compiler workaround the retired
  copy carried (git-applied at build time) for an AffineScript WASM-codegen
  bug: imported enum constructors weren't linked (`Codegen.UnboundVariable`
  on `use M::{Some, None}` compiled to WASM). **Almost certainly already
  fixed upstream** — `hyperpolymath/affinescript#602` ("fix(codegen): #138 —
  link imported enum constructors into codegen", merged 2026-06-20) matches
  this fix nearly verbatim and landed one day before this patch's local
  commit date. This repo's own `.affine` files (28 of them, more than the
  retired copy's 11) never needed this patch, which is consistent with the
  fix already being present in whatever AffineScript version this repo
  builds against. Kept for reference only — do not apply without first
  confirming (e.g. by trying to compile a cross-module enum-constructor
  import to WASM) that the bug has actually resurfaced.
