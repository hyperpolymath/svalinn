= Production Deployment Guide
:toc: preamble
:toclevels: 3
:icons: font

Complete guide for deploying Svalinn Edge Gateway in production environments.

== Prerequisites

=== Runtime Requirements

* **Deno** >= 2.0 (runtime for compiled ReScript output)
* **Vordr** container runtime instance (accessible via MCP protocol)
* **TLS certificates** for HTTPS (Let's Encrypt, internal CA, or self-signed for dev)
* **Linux** host (Fedora, Ubuntu, Debian, etc.)

=== Build Requirements

* **ReScript** compiler (installed via `deno install`)
* **Just** command runner (optional but recommended)
* **Podman** >= 4.0 for container builds

== Container Deployment (Recommended)

=== Build the Image

The repository ships a multi-stage `Containerfile` using Chainguard wolfi-base
images. Stage 1 compiles ReScript; stage 2 produces a minimal runtime image.

[source,bash]
----
podman build -f Containerfile -t svalinn:latest .
----

=== Run with Podman

[source,bash]
----
podman run -d -p 8000:8000 --name svalinn \
  -e VORDR_ENDPOINT=http://vordr:8080 \
  -e AUTH_ENABLED=true \
  -e TLS_CERT_FILE=/certs/server.crt \
  -e TLS_KEY_FILE=/certs/server.key \
  -v /etc/svalinn/certs:/certs:ro,Z \
  svalinn:latest
----

=== Production Compose (selur-compose)

[source,yaml]
----
version: "3"
services:
  svalinn:
    image: svalinn:latest
    ports:
      - "8000:8000"
    environment:
      SVALINN_PORT: "8000"
      SVALINN_HOST: "0.0.0.0"
      VORDR_ENDPOINT: "http://vordr:8080"
      ROKUR_ENDPOINT: "http://rokur:9090"
      ROKUR_GATE_ENABLED: "true"
      AUTH_ENABLED: "true"
      AUTH_METHODS: "oidc,apikey"
      TLS_CERT_FILE: "/certs/server.crt"
      TLS_KEY_FILE: "/certs/server.key"
      ALLOWED_ORIGINS: "https://dashboard.example.com"
      RATE_LIMIT_WINDOW_MS: "60000"
      RATE_LIMIT_MAX_REQUESTS: "200"
      LOG_LEVEL: "info"
    volumes:
      - /etc/svalinn/certs:/certs:ro
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "deno", "eval", "const r = await fetch('http://localhost:8000/healthz'); Deno.exit(r.ok ? 0 : 1)"]
      interval: 15s
      timeout: 5s
      retries: 3
      start_period: 10s
    depends_on:
      - vordr
      - rokur
----

== Bare Metal Deployment

=== Direct Deno Execution

[source,bash]
----
# Build ReScript source
cd /opt/svalinn
deno install
cd src && ./node_modules/.bin/rescript build && cd ..

# Run with production settings
SVALINN_PORT=8000 \
SVALINN_HOST=0.0.0.0 \
VORDR_ENDPOINT=http://vordr.internal:8080 \
ROKUR_ENDPOINT=http://rokur.internal:9090 \
AUTH_ENABLED=true \
TLS_CERT_FILE=/etc/svalinn/certs/server.crt \
TLS_KEY_FILE=/etc/svalinn/certs/server.key \
deno run \
  --allow-net \
  --allow-env \
  --allow-read \
  src/Main.res.js
----

=== Systemd Service

Create `/etc/systemd/system/svalinn.service`:

[source,ini]
----
[Unit]
Description=Svalinn Edge Gateway
After=network.target vordr.service
Requires=vordr.service

[Service]
Type=simple
User=svalinn
Group=svalinn
WorkingDirectory=/opt/svalinn
ExecStart=/usr/bin/deno run \
  --allow-net --allow-env --allow-read \
  src/Main.res.js
Restart=on-failure
RestartSec=5s

# Environment file keeps secrets out of unit definitions
EnvironmentFile=/etc/svalinn/svalinn.env

# Security hardening
NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=strict
ProtectHome=true
ReadOnlyPaths=/etc/svalinn/certs

[Install]
WantedBy=multi-user.target
----

Enable and start:

[source,bash]
----
sudo systemctl daemon-reload
sudo systemctl enable --now svalinn
sudo systemctl status svalinn
----

== TLS Configuration

Svalinn supports native TLS via Deno's built-in HTTPS server. When both
`TLS_CERT_FILE` and `TLS_KEY_FILE` are set, the gateway serves HTTPS directly
without requiring a reverse proxy.

=== Let's Encrypt (Production)

[source,bash]
----
# Provision certificates with certbot
sudo certbot certonly --standalone -d api.example.com

# Point Svalinn at the live certs
export TLS_CERT_FILE=/etc/letsencrypt/live/api.example.com/fullchain.pem
export TLS_KEY_FILE=/etc/letsencrypt/live/api.example.com/privkey.pem
----

NOTE: Set up a cron job or systemd timer for `certbot renew`. After renewal,
restart or send SIGHUP to the Svalinn process to pick up new certificates.

=== Self-Signed (Development)

[source,bash]
----
# Generate a self-signed cert for local development
openssl req -x509 -newkey rsa:4096 -sha256 -days 365 -nodes \
  -keyout dev-key.pem -out dev-cert.pem \
  -subj "/CN=localhost" \
  -addext "subjectAltName=DNS:localhost,IP:127.0.0.1"

export TLS_CERT_FILE=./dev-cert.pem
export TLS_KEY_FILE=./dev-key.pem
----

=== Reverse Proxy TLS Termination (Alternative)

If you prefer TLS termination at a reverse proxy (Nginx, Caddy), leave
`TLS_CERT_FILE` and `TLS_KEY_FILE` unset. Svalinn will serve plain HTTP and
the proxy handles HTTPS.

[source,nginx]
----
upstream svalinn {
    server 127.0.0.1:8000;
}

server {
    listen 443 ssl http2;
    server_name api.example.com;

    ssl_certificate /etc/ssl/certs/api.example.com.crt;
    ssl_certificate_key /etc/ssl/private/api.example.com.key;

    location / {
        proxy_pass http://svalinn;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /healthz {
        proxy_pass http://svalinn;
        access_log off;
    }
}
----

== Authentication Setup

=== OIDC Provider Configuration

[source,bash]
----
export AUTH_ENABLED=true
export AUTH_METHODS=oidc,apikey

# OIDC provider (e.g. Keycloak, Authentik, Authelia)
export OIDC_ISSUER=https://auth.example.com/realms/svalinn
export OIDC_CLIENT_ID=svalinn-prod
export OIDC_CLIENT_SECRET=<secret-from-provider>
export OIDC_AUTH_ENDPOINT=https://auth.example.com/realms/svalinn/protocol/openid-connect/auth
export OIDC_TOKEN_ENDPOINT=https://auth.example.com/realms/svalinn/protocol/openid-connect/token
export OIDC_USERINFO_ENDPOINT=https://auth.example.com/realms/svalinn/protocol/openid-connect/userinfo
export OIDC_JWKS_URI=https://auth.example.com/realms/svalinn/protocol/openid-connect/certs
export OIDC_REDIRECT_URI=https://api.example.com/auth/callback
export OIDC_SCOPES=openid,profile,email
----

=== API Key Management

API keys are provided as a comma-separated list. Generate cryptographically
strong keys and rotate regularly.

[source,bash]
----
# Generate a new API key
KEY=$(openssl rand -hex 32)
echo "sk_prod_${KEY}"

# Set multiple keys (allows rotation without downtime)
export API_KEYS=sk_prod_<current>,sk_prod_<previous>
----

Clients authenticate via the `Authorization` header:

[source,bash]
----
curl -H "Authorization: Bearer sk_prod_abc123..." https://api.example.com/api/v1/containers
# or
curl -H "X-API-Key: sk_prod_abc123..." https://api.example.com/api/v1/containers
----

== Rate Limiting

Svalinn applies a per-IP sliding-window rate limiter to all endpoints.

=== Tuning for Production Traffic

[cols="1,2,2",options="header"]
|===
|Profile |Window |Max Requests

|Low traffic (internal)
|`RATE_LIMIT_WINDOW_MS=60000`
|`RATE_LIMIT_MAX_REQUESTS=100`

|Medium traffic
|`RATE_LIMIT_WINDOW_MS=60000`
|`RATE_LIMIT_MAX_REQUESTS=500`

|High traffic (public API)
|`RATE_LIMIT_WINDOW_MS=60000`
|`RATE_LIMIT_MAX_REQUESTS=2000`

|Burst-friendly
|`RATE_LIMIT_WINDOW_MS=10000`
|`RATE_LIMIT_MAX_REQUESTS=200`
|===

When a client exceeds the limit, Svalinn returns `429 Too Many Requests`.

== Health Checks

=== Endpoints for Orchestrators

[cols="1,2,2",options="header"]
|===
|Endpoint |Purpose |Behaviour

|`/healthz`
|Liveness probe
|Returns `200` with JSON status; checks Vordr connectivity and reports `healthy` or `degraded`

|`/readyz`
|Readiness probe
|Returns `200` if Vordr is reachable, `503` if Vordr is unavailable

|`/metrics`
|Prometheus scrape
|Returns `200` with text exposition format
|===

=== Podman / Kubernetes Health Check

[source,bash]
----
# Podman
podman run --health-cmd='deno eval "const r = await fetch(\"http://localhost:8000/healthz\"); Deno.exit(r.ok ? 0 : 1)"' \
  --health-interval=15s --health-timeout=5s --health-retries=3 ...

# Kubernetes
livenessProbe:
  httpGet:
    path: /healthz
    port: 8000
  initialDelaySeconds: 10
  periodSeconds: 15
readinessProbe:
  httpGet:
    path: /readyz
    port: 8000
  initialDelaySeconds: 5
  periodSeconds: 10
----

== Security Checklist

Before going to production, verify every item:

* [ ] `AUTH_ENABLED=true` -- authentication middleware active
* [ ] `ALLOWED_ORIGINS` set to specific domains (no wildcard `*`)
* [ ] TLS enabled -- either `TLS_CERT_FILE`/`TLS_KEY_FILE` set or reverse proxy terminating TLS
* [ ] `RATE_LIMIT_MAX_REQUESTS` tuned for expected traffic (not left at default 100 for high-traffic)
* [ ] `ROKUR_GATE_ENABLED=true` -- container-start authorization active
* [ ] `ROKUR_API_TOKEN` set to a strong, rotated token
* [ ] `API_KEYS` use cryptographically strong random values (>= 32 bytes hex)
* [ ] `OIDC_CLIENT_SECRET` stored in a secrets manager, not in plaintext env files
* [ ] `LOG_LEVEL=info` or `warn` (not `debug` in production -- may leak sensitive data)
* [ ] Container runs as non-root user (the shipped Containerfile already does this)
* [ ] Network segmentation: Vordr and Rokur not exposed to the public internet
* [ ] HSTS header enabled (Svalinn sets this by default via security headers middleware)
* [ ] No wildcard `*` in `ALLOWED_ORIGINS`

== Troubleshooting

=== Common Issues

[cols="1,2,2",options="header"]
|===
|Symptom |Likely Cause |Fix

|`503 Service Unavailable` on `/readyz`
|Vordr not reachable
|Verify `VORDR_ENDPOINT` is correct and Vordr is running. Check network/firewall rules.

|`401 Unauthorized` on API calls
|Auth enabled, missing or invalid token
|Provide a valid JWT or API key via `Authorization: Bearer <token>` or `X-API-Key: <key>` header.

|`409 Conflict` on container start
|Rokur denied the container start
|Check the `rokur` field in the response for the denial reason. Review Rokur policy configuration.

|`429 Too Many Requests`
|Rate limit exceeded
|Wait for the window to reset, or increase `RATE_LIMIT_MAX_REQUESTS`.

|`400 Bad Request` on `/api/v1/run`
|Request body fails JSON schema validation
|Check the `details` array in the response for specific validation errors.

|TLS handshake failure
|Cert/key mismatch or expired certificate
|Verify cert and key match: `openssl x509 -noout -modulus -in cert.pem \| md5sum` vs `openssl rsa -noout -modulus -in key.pem \| md5sum`. Renew if expired.

|`500 Internal Server Error`
|Unhandled runtime error
|Set `LOG_LEVEL=debug` temporarily and inspect structured JSON logs for the `error` field.

|Slow responses (>1s)
|Vordr MCP timeout or network latency
|Increase `VORDR_TIMEOUT`. Check Vordr health. Consider co-locating Svalinn and Vordr.

|Gateway starts but no HTTPS
|`TLS_CERT_FILE` or `TLS_KEY_FILE` not set (both required)
|Set both env vars. Svalinn logs `TLS disabled` vs `TLS enabled` at startup -- check the log output.

|Rokur authorization retries exhaust
|Rokur returning 5xx errors
|Check Rokur health. Increase `ROKUR_RETRY_COUNT` (max 5) and `ROKUR_TIMEOUT_MS` if transient.
|===

=== Debug Mode

[source,bash]
----
# Enable verbose structured logging
export LOG_LEVEL=debug

# Check Vordr connectivity directly
curl -s http://localhost:8080/health | jq .

# Check Rokur connectivity
curl -s http://localhost:9090/health | jq .

# Inspect Svalinn startup metadata
journalctl -u svalinn --since "5 min ago" | jq 'select(.message == "Starting Svalinn Gateway")'
----

=== Log Analysis

Svalinn emits structured JSON on stdout. Parse with `jq`:

[source,bash]
----
# Follow logs
journalctl -u svalinn -f | jq .

# Errors only
journalctl -u svalinn | jq 'select(.level == "ERROR")'

# Slow requests (>500ms)
journalctl -u svalinn | jq 'select(.metadata.duration_ms > 500)'

# Rokur denials
journalctl -u svalinn | jq 'select(.message | contains("Rokur denied"))'
----

== Upgrading

[source,bash]
----
# 1. Pull latest source
git pull origin main

# 2. Rebuild container image
podman build -f Containerfile -t svalinn:latest .

# 3. Rolling restart (zero-downtime if behind a load balancer)
podman stop svalinn
podman rm svalinn
podman run -d -p 8000:8000 --name svalinn \
  --env-file /etc/svalinn/svalinn.env \
  -v /etc/svalinn/certs:/certs:ro,Z \
  svalinn:latest
----
