== Core Claims

=== Claim 1: Hono-Based HTTP Gateway with JSON Schema Validation and MCP/JSON-RPC Delegation

*From README:* "Svalinn is a Deno-based HTTP gateway that validates container operation requests and delegates to Vörðr. It provides REST API access, JSON Schema validation, policy enforcement, OAuth2/JWT authentication, and a Compose-compatible orchestration CLI (svalinn-compose)."

*Evidence:* The gateway is built with Hono (web framework for Deno/Workers) at `src/main.ts`. Request validation uses JSON Schema files at `spec/schemas/` (gateway-*.json, container-*.json) loaded dynamically. The gateway routes are defined in `src/gateway/` with middleware at `src/validation/` applying schema checks before delegating to Vörðr. The Vörðr integration at `src/vordr/` implements an MCP/JSON-RPC client that forwards validated requests to the container engine. Test coverage includes schema validation tests at `src/tests/`. The REST API endpoints listed in the README (GET /v1/containers, POST /v1/run, POST /v1/verify, etc.) are implemented in the gateway routing layer. **Caveat:** Vörðr integration depends on Vörðr being running; svalinn is purely a validation gateway and does not provide container runtime guarantees itself.

=== Claim 2: Compose-Compatible Orchestration CLI with Security Extensions

*From README:* "svalinn-compose: Compose-compatible multi-container orchestration CLI... Configurable allow/deny rules per operation, OAuth2/OIDC + JWT token validation middleware."

*Evidence:* The `svalinn-compose` CLI implementation at `src/compose/` provides Docker Compose compatibility with security extensions. The compose file parser at `src/compose/parser.ts` accepts standard docker-compose.yaml format and additional `x-svalinn:` sections for policy overrides. The orchestrator at `src/compose/orchestrator.ts` delegates to the Svalinn HTTP gateway for each container operation. Policy enforcement at `src/policy/` evaluates rules per operation type (run, stop, delete). Authentication middleware at `src/auth/` validates OAuth2 tokens and JWT claims. The tool supports commands: `up`, `down`, `ps`, `logs`, `scale`, `config` with security checks at each step. **Caveat:** This is a proof-of-concept; production orchestration may require additional features (rollback, health checks, secrets management).
